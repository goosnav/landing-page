DOCUMENTED DECISION

The source specification requires page shells in site/ and shared assets
in assets/. GitHub Pages publishes HTML from the artifact root. The repo
therefore uses tools/build-site.mjs to assemble dist/ for deployment.

This was chosen because it is:

- lower-bloat than adding a framework or bundler
- more stable than duplicating source files
- more reusable than collapsing the repo structure around Pages limits
